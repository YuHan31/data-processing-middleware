## 一、项目概述

### 1.1 项目背景

随着企业数据量级的爆炸式增长，数据清洗与预处理已成为数据工程链路中不可或缺的一环。然而，传统 ETL 工具（如 Kettle、Informatica）部署复杂、学习成本高，中小型团队往往缺乏专职数据工程师。本项目旨在构建一个**轻量级、零运维、即开即用**的数据处理中间件系统，为非技术人员提供可视化、操作友好的数据清洗与导出能力。

### 1.2 项目目标

- 支持多种数据格式（CSV、Excel、JSON）的上传、解析与处理
- 提供 10 种可配置的数据清洗规则，覆盖格式化、规范化、安全脱敏三大类别
- 支持清洗规则动态编排，按执行顺序组合使用
- 提供任务全生命周期管理（创建、启动、停止、删除）
- 支持数据处理前后的可视化对比
- 实现用户鉴权与角色权限控制（普通用户 / 管理员）
- 提供管理后台，供管理员监控系统运行状态

### 1.3 技术选型

| 技术类别 | 选型方案 | 版本 | 说明 |
|---|---|---|---|
| 后端框架 | Spring Boot | 2.7.18 | 主流 Java Web 框架 |
| Java 版本 | Java | 17 | LTS 版本，现代化语法支持 |
| ORM 框架 | MyBatis-Plus | 3.5.3.1 | 简化 CRUD，提升开发效率 |
| 数据库 | MySQL | 8.0 | 关系型数据持久化 |
| Excel 处理 | Apache POI | 5.2.3 | 支持 .xlsx 文件读写 |
| JSON 处理 | Jackson | 内置 | Spring Boot 默认 JSON 库 |
| API 文档 | Springdoc OpenAPI | 1.6.15 | 自动生成 Swagger UI |
| 密码加密 | Spring Security Crypto | 内置 | BCrypt 哈希算法 |
| 构建工具 | Maven | 3.x | 依赖管理与打包 |
| 前端 | HTML + JavaScript（原生） | — | 前后端分离，REST API 交互 |

## 三、核心功能模块

### 3.1 数据解析模块（Parser）

**设计模式：工厂模式 + 策略模式**

解析模块负责将用户上传的原始文件（CSV、Excel、JSON）解析为统一的内存数据结构（`ProcessedData`）。

| 组件 | 职责 |
|---|---|
| `IDataParser` | 解析器统一接口，定义 `parse()` 方法 |
| `CsvParser` | 实现 CSV 文件解析（支持大文件流式读取） |
| `ExcelParser` | 实现 Excel（.xlsx）解析（Apache POI） |
| `JsonParser` | 实现 JSON 解析（Jackson） |
| `ParserFactory` | 根据文件类型动态选择解析器，无需修改调用方代码 |

**解析流程：**
```
用户上传文件 → 根据文件扩展名选择 Parser
→ ParserFactory.getParser(type)
→ 调用 parse() 方法
→ 返回统一的 ProcessedData 对象
```

### 3.2 数据清洗模块（Cleaner）— 核心亮点

**设计模式：责任链模式 + 工厂模式**

清洗模块是本系统的核心特色，采用**动态清洗链（CleanerChain）** 架构。用户可以在启动任务时自由选择清洗规则及其执行顺序，系统按链式依次执行。

#### 3.2.1 清洗规则分类

系统预置了 **10 种清洗规则**，分为三大类别：

**FORMAT（格式化）— 基础级别**

| 规则代码 | 名称 | 说明 |
|---|---|---|
| `TRIM` | 去除空格 | 去除字段值的首尾空格 |
| `TO_LOWER` | 转小写 | 将文本字段统一转为小写 |
| `TO_UPPER` | 转大写 | 将文本字段统一转为大写 |

**NORMALIZE（规范化）— 基础级别**

| 规则代码 | 名称 | 说明 |
|---|---|---|
| `REMOVE_NULL` | 删除空值 | 删除字段值为空或 null 的记录 |
| `REMOVE_EMPTY_ROW` | 删除空行 | 删除所有字段均为空的记录 |
| `DEDUPLICATE` | 去除重复 | 删除完全重复的记录 |
| `NORMALIZE_DATE` | 日期标准化 | 将日期统一为标准格式 `yyyy-MM-dd` |

**SECURITY（安全脱敏）— 进阶级别**

| 规则代码 | 名称 | 说明 |
|---|---|---|
| `DATA_MASK` | 数据脱敏 | 对敏感数据进行通用脱敏处理 |
| `PHONE_MASK` | 手机号脱敏 | 显示前三位 + `****` + 后四位（如 `138****1234`） |
| `EMAIL_MASK` | 邮箱脱敏 | 邮箱用户名部分脱敏（如 `j***n@example.com`） |

#### 3.2.2 清洗链执行流程

```
任务启动 → CleanerChain.clean(processedData, taskContext)
          ↓
  1. 根据 taskId 查询 task_clean_rule 关联表
  2. 按 exec_order 升序排列规则
  3. 遍历规则列表：
       CleanerFactory.get(ruleCode) → 获取 IDataCleaner 实例
       → cleaner.clean(processedData, taskContext)
       → 异常则抛出 DataCleanException，链式中断
  4. 全部规则执行完成 → 进入导出阶段
```

### 3.3 数据导出模块（Exporter）

**设计模式：工厂模式 + 策略模式**

清洗完成后，支持将数据导出为多种格式：

| 导出器 | 支持格式 | 说明 |
|---|---|---|
| `CsvExporter` | .csv | 逗号分隔值文件，通用兼容 |
| `ExcelExporter` | .xlsx | Excel 格式，适合人工查阅 |
| `JsonExporter` | .json | 结构化 JSON，适合程序处理 |

### 3.4 任务管理模块

**完整任务生命周期：**

```
[创建任务] → [上传文件] → [选择清洗规则] → [启动任务]
                                              ↓
[完成 FINISHED] ← [导出结果] ← [数据清洗] ← [数据解析 PARSING]
    ↕                                              ↑
[失败 FAILED]                              [运行中 RUNNING]
    ↑
[停止 STOPPED]
```

| 功能 | 接口 | 说明 |
|---|---|---|
| 创建任务 | `POST /api/task/create` | 初始化任务上下文 |
| 启动任务 | `POST /api/task/start/{taskId}` | 传入清洗规则列表，开始异步处理 |
| 查询状态 | `GET /api/task/status/{taskId}` | 查询当前任务状态和进度百分比 |
| 任务列表 | `GET /api/task/list` | 分页获取当前用户任务列表（page 从 1 开始） |
| 查询进度 | `GET /api/task/progress/{taskId}` | 获取实时进度和流程节点（含 5 个阶段） |
| 停止任务 | `POST /api/task/stop/{taskId}` | 中断正在执行的任务 |
| 删除任务 | `DELETE /api/task/{taskId}` | 删除指定任务及关联数据 |
| 数据对比 | `GET /api/task/compare/{taskId}` | 清洗前后数据可视化对比（分页+搜索） |
| 对比统计 | `GET /api/task/compare/{taskId}/stats` | 变化率、各字段修改次数统计 |
| 按字段查看 | `GET /api/task/compare/{taskId}/field/{fieldName}` | 查看特定字段的所有变化 |

### 3.5 数据对比模块

本模块实现清洗前后数据的一对一对比，类似 Git 版本对比体验：

- **全量对比**：按分页返回所有原始值 vs 清洗后值
- **仅变化数据**：通过 `onlyChanged=true` 筛选出有变化的行
- **关键词搜索**：支持按字段名或字段值搜索
- **字段级对比**：通过 `field/{fieldName}` 查看特定字段变化
- **统计概览**：返回总记录数、变化率、各字段修改次数

## 五、技术亮点与设计模式

### 5.1 工厂模式 — Parser / Cleaner / Exporter

三个核心处理模块均采用工厂模式实现动态组件选择，新增一种组件时**无需修改已有代码**，只需注册到工厂即可，符合开闭原则：

```
用户上传 test.csv
  → ParserFactory.getParser("csv") → CsvParser
  → 数据流入 CleanerChain

用户选择 ["TRIM", "REMOVE_NULL", "PHONE_MASK"]
  → CleanerFactory.get("TRIM")     → TrimCleaner
  → CleanerFactory.get("REMOVE_NULL") → RemoveNullCleaner
  → CleanerFactory.get("PHONE_MASK") → PhoneMaskCleaner
  → 按 exec_order 顺序执行，形成清洗链

用户选择输出格式 "excel"
  → ExporterFactory.get("excel") → ExcelExporter
```

### 5.2 责任链模式 — CleanerChain

清洗链按照用户配置的执行顺序，依次将数据交给每个 Cleaner 处理：

- 每个 Cleaner 只负责一种清洗逻辑（**单一职责**）
- 链的顺序可动态配置（**灵活性**）
- 单个 Cleaner 失败不影响其他规则（**容错性**）

### 5.3 异步任务执行

任务执行采用 Spring `@Async` + 自定义线程池：

```java
// TaskThreadPoolConfig.java
核心线程数：2          // CPU 核数 × 2
最大线程数：4          // CPU 核数 × 4
队列容量：100

// TaskExecutor.java
@Async("taskExecutor")
public void executeTask(String taskId, String outputFormat,
                        String outputPath, List<String> rules)
```

优势：HTTP 请求不阻塞，用户发起任务后立即返回 `200`，任务在后台线程池异步执行，前端可通过轮询 `/api/task/progress/{taskId}` 获取进度。

### 5.4 统一响应与全局异常处理

所有接口返回统一格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

`GlobalExceptionHandler` 统一捕获并处理 `DataCleanException`、`DataParseException`、`FileProcessException` 等所有未处理异常，返回结构化错误响应，避免原始异常堆栈泄露。

### 5.5 认证与会话管理

- 登录成功后用户信息（`userId`）存入 `HttpSession`
- `AuthInterceptor` 拦截所有 `/api/task/*`、`/api/file/*` 等接口，验证 Session 中是否存在 `userId`
- 密码使用 **BCrypt** 哈希算法存储（不可逆，比 MD5/SHA 更安全）
- 管理员接口由 `AdminController` 独立管理，通过 `role = 'ADMIN'` 区分

## 十一、项目亮点总结

1. **三大设计模式贯穿核心**：工厂模式（Parser/Exporter/Cleaner）+ 策略模式 + 责任链模式（CleanerChain），代码扩展性极佳

2. **动态清洗链编排**：用户可自由选择 10 种清洗规则并自定义执行顺序，真正实现"所见即所得"的配置化清洗

3. **异步任务执行**：后台线程池异步处理，HTTP 请求不阻塞，支持任务停止与进度实时查询

4. **数据可视化对比**：类似 Git Diff 的前后数据对比体验，支持关键词搜索、字段级过滤和变化统计

5. **完整用户体系**：BCrypt 加密 + 图形验证码 + Session 会话 + 角色权限（USER/ADMIN）

6. **生产级异常处理**：全局统一异常捕获，结构化错误响应，避免堆栈信息泄露

7. **开箱即用的 API 文档**：集成 Springdoc OpenAPI，访问 `/swagger-ui.html` 即可交互式测试所有接口

8. **插件式架构**：新增解析器/清洗器/导出器只需实现接口并注册，无需修改已有代码
